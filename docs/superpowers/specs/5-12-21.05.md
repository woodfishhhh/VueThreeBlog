# Works 3D Interaction Tuning

## Summary

- 把 night 默认超立方体放大到 day 莫比乌斯环同级体
  量。
- WebGL 作品卡片整体放大，并保留围绕中心几何体公
  转。
- Works 模式下中心几何体只作为视觉锚点：禁用点击沉
  浸、禁用 hover 变色。
- 卡片点击改为“抓牌”：按下拉到镜头前，松手悬浮
  500ms，之后回到轨道继续转。

## Key Changes

- ThreeSceneCanvas.vue
  - 拆出 HYPERCUBE_SCENE_SCALE = 2、
    MOBIUS_SCENE_SCALE = 2，两种主题主几何体使用
    同级 scale。
  - Works 模式 raycast 优先只处理卡片；未点中卡
    片时直接返回，不进入几何体 focus。
  - hover 状态拆分为 cardHover/grab 与
    geometryHover；Works 模式不再对几何体做
    hover raycast/color lerp。
  - pointerdown 命中卡片 → grab；pointerup/
    pointercancel/lostpointercapture → release；
    同时 suppress 后续 canvas click，避免触发背
    景 step。
- works-orbit-cards.ts
  - 卡片尺寸调到 4.25 x 2.4，轨道 scale 改为 0.9
    - frontness \* 0.3。
  - 轨道半径扩大：radiusX clamp(width/185, 6.4,
    8.4)，radiusY clamp(width/700, 1.35, 2.1)，
    radiusZ clamp(width/460, 2.6, 3.7)。
  - 新增 grab 状态：grab(slug) /
    release(elapsed) / clearInteraction()；
    update() 接收 delta。
  - grabbed/released 卡片移动到 camera 前方约
    5.2 units，scale 1.45，renderOrder 最高；
    release 后保持 500ms 再平滑回轨道。
  - 桌面 WebGL 卡片点击只抓取，不打开链接；移除/
    弱化纹理里的 Live/GitHub 按钮假 affordance。
- useHypercube.ts
  - hit sphere 半径从 1.5 放大到 3.1，覆盖整个放
    大后的超立方体。

## Interfaces / Types

- WorksOrbitCards.update(options) 增加 delta。
- WorksOrbitCards 增加：
  - grab(slug: string): void
  - release(elapsed: number): void
  - clearInteraction(): void
  - isInteracting(): boolean
  - useHypercube 校验 hit sphere radius 覆盖扩大
    值。
  - scene/interaction 逻辑测试：Works 模式不调用
    几何体 focus；卡片 grab/release 500ms 后恢复
    orbit。
- Verification：
  - npm test
  - npm run typecheck
  - npm run build
  - Browser smoke：/works 1440 night/day、390
    mobile；确认卡片更大、中心几何体不 hover 变
    色、不进入沉浸，抓牌动画可见，console 无
    error/warn。

## Assumptions

- 已按你的选择锁定：桌面 WebGL 卡片点击只触发抓取
  动效，不自动打开新页面。
- 移动端 Works 列表 fallback 不改，仍保留项目链
  接。
